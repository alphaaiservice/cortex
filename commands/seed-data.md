# Seed Data Generator

Generate realistic seed/test data for: **$ARGUMENTS**

Parse optional arguments from $ARGUMENTS:
- `--count=N` — Number of records per model (default: 10)
- `--model=ModelName` — Generate data for a specific model only (default: all models)
- `--format=sql|json|python` — Output format preference (default: python)
- `--reset` — Include reset/truncate logic in the seeder script

## Step 1: Discover Data Models

Scan the entire project to locate and parse all data models. Build a complete understanding of the data layer before generating anything.

### 1a: Find SQLAlchemy Models

```bash
# Search for SQLAlchemy model classes
```

Use Grep to find patterns:
- `class .*(Base):` or `class .*(DeclarativeBase)` — SQLAlchemy declarative models
- `class .*(db.Model):` — Flask-SQLAlchemy models
- `Column(` — SQLAlchemy column definitions
- `mapped_column(` — SQLAlchemy 2.0 mapped columns
- `relationship(` — SQLAlchemy relationships
- `ForeignKey(` — Foreign key references
- `__tablename__` — Table name declarations

### 1b: Find MongoDB / Pydantic Models

Use Grep to find patterns:
- `class .*(BaseModel):` — Pydantic models
- `class .*(Document):` — Beanie/ODM document models
- `class .*(MongoModel):` — Custom MongoDB models
- `Collection` or `collection_name` references

### 1c: Find Django Models (if applicable)

Use Grep to find patterns:
- `class .*(models.Model):` — Django ORM models
- `models.CharField`, `models.IntegerField`, etc.

### 1d: Parse Each Model

For every discovered model, extract:

1. **Model name** — The class name (e.g., `User`, `Order`, `Product`)
2. **Table/collection name** — From `__tablename__` or class name convention
3. **Fields** — Each column/field with:
   - Field name (e.g., `email`, `full_name`, `price`)
   - Data type (e.g., `String`, `Integer`, `Float`, `Boolean`, `DateTime`, `Enum`, `JSON`, `UUID`)
   - Constraints:
     - `nullable` (True/False)
     - `unique` (True/False)
     - `default` value
     - `server_default` value
     - `max_length` / `String(255)` length
   - Primary key flag
   - Index flag
4. **Foreign keys** — Which field references which table (e.g., `user_id -> users.id`)
5. **Relationships** — `relationship()` declarations with back_populates
6. **Enum types** — Any Python Enum classes used by model fields

### 1e: Build Dependency Graph

Using foreign key references, build a directed graph of model dependencies:

```
Example:
  User → (no dependencies, seed first)
  SubscriptionPlan → (no dependencies, seed first)
  UserSubscription → depends on [User, SubscriptionPlan]
  Order → depends on [User]
  OrderItem → depends on [Order, Product]
  Payment → depends on [Order, User]
```

Sort models in **topological order** — models with no foreign keys first, then models that depend on already-seeded models. This ensures referential integrity when inserting.

If circular dependencies are detected, note them and use a two-pass strategy (insert without FK first, then update FK references).

### 1f: Detect Application Domain

Analyze model names and fields to infer the application domain:

| Domain | Indicators |
|--------|-----------|
| **E-commerce** | Product, Category, Order, Cart, Review, Inventory, Coupon |
| **SaaS** | Organization, Team, Subscription, Invoice, Plan, Feature, Tenant |
| **Social** | Post, Comment, Like, Follow, Feed, Story, Message |
| **Education** | Course, Lesson, Enrollment, Quiz, Assignment, Grade, Student |

Use the detected domain to generate **contextually appropriate** fake data (not random gibberish).

### 3b: Seeder Logic

The seeder must:

1. **Parse CLI arguments** (`--count`, `--model`, `--reset`, `--format`)
2. **Connect to the database** — detect whether the project uses SQLAlchemy async sessions, sync sessions, or PyMongo
3. **Reset if requested** — truncate tables in reverse dependency order (children first, parents last) to avoid FK constraint violations
4. **Seed in dependency order** — insert parent models before child models
5. **Handle foreign keys** — when creating a child record (e.g., `Order`), pick a random existing parent ID from the already-seeded parent table (e.g., `User`)
6. **Track inserted IDs** — maintain a dict of `{model_name: [list_of_inserted_ids]}` for FK resolution
7. **Create default accounts**:
   - Admin user: `admin@example.com` / `Admin@123`
   - Test user: `testuser@example.com` / `Test@123`
8. **Create default subscription plans** (if SubscriptionPlan or Plan model exists):
   - Free plan: $0/month, basic features
   - Pro plan: $29/month, advanced features
   - Enterprise plan: $99/month, all features
9. **Create default credit point balances** (if CreditPoint or Wallet model exists):
   - Admin: 10,000 credits
   - Test user: 500 credits
10. **Print summary** — table of model name, records inserted, time taken

### 3c: SQLAlchemy Seeder Pattern

For SQLAlchemy-based projects, generate:

```python
async def seed_sqlalchemy(session, models_to_seed, count, reset):
    """Seed MySQL/PostgreSQL via SQLAlchemy async session."""
    from sqlalchemy import text

    if reset:
        logger.info("Resetting database (truncating tables)...")
        await session.execute(text("SET FOREIGN_KEY_CHECKS = 0"))
        for model_name in reversed(SEED_ORDER):
            table_name = MODEL_TABLE_MAP[model_name]
            await session.execute(text(f"TRUNCATE TABLE `{table_name}`"))
            logger.info(f"  Truncated: {table_name}")
        await session.execute(text("SET FOREIGN_KEY_CHECKS = 1"))
        await session.commit()

    inserted_ids = {}

    for model_name in SEED_ORDER:
        if models_to_seed and model_name not in models_to_seed:
            continue

        factory = FACTORY_MAP[model_name]
        model_class = MODEL_CLASS_MAP[model_name]
        start = time.time()

        records = []
        for i in range(count):
            overrides = {}
            # Resolve foreign keys from already-inserted parent IDs
            for fk_field, parent_model in FK_MAP.get(model_name, {}).items():
                if parent_model in inserted_ids and inserted_ids[parent_model]:
                    overrides[fk_field] = random.choice(inserted_ids[parent_model])

            data = factory.create(**overrides)
            obj = model_class(**data)
            session.add(obj)
            records.append(obj)

        await session.flush()  # Get auto-generated IDs
        inserted_ids[model_name] = [r.id for r in records]
        elapsed = round(time.time() - start, 2)
        logger.info(f"  Seeded {count} x {model_name} ({elapsed}s)")

    await session.commit()
    return inserted_ids
```

### 3d: MongoDB Seeder Pattern

For MongoDB/PyMongo-based projects, generate:

```python
def seed_mongodb(db, models_to_seed, count, reset):
    """Seed MongoDB via PyMongo."""
    if reset:
        logger.info("Resetting MongoDB collections...")
        for model_name in reversed(SEED_ORDER):
            collection_name = MODEL_COLLECTION_MAP[model_name]
            db[collection_name].delete_many({})
            logger.info(f"  Cleared: {collection_name}")

    inserted_ids = {}

    for model_name in SEED_ORDER:
        if models_to_seed and model_name not in models_to_seed:
            continue

        factory = FACTORY_MAP[model_name]
        collection_name = MODEL_COLLECTION_MAP[model_name]
        start = time.time()

        documents = []
        for i in range(count):
            overrides = {}
            for ref_field, parent_model in REF_MAP.get(model_name, {}).items():
                if parent_model in inserted_ids and inserted_ids[parent_model]:
                    overrides[ref_field] = random.choice(inserted_ids[parent_model])

            doc = factory.create(**overrides)
            documents.append(doc)

        result = db[collection_name].insert_many(documents)
        inserted_ids[model_name] = list(result.inserted_ids)
        elapsed = round(time.time() - start, 2)
        logger.info(f"  Seeded {count} x {model_name} ({elapsed}s)")

    return inserted_ids
```

## Step 6: Domain-Aware Data Generation

Based on the domain detected in Step 1f, generate contextually appropriate data. Do NOT generate generic lorem ipsum where domain-specific data would be more realistic.

### E-commerce Domain

If models include `Product`, `Category`, `Order`, etc.:
- Products: Use real-sounding product names ("Wireless Bluetooth Headphones", "Organic Cotton T-Shirt")
- Categories: Use realistic hierarchies ("Electronics > Audio > Headphones")
